# 买入策略优化对比

## 旧方案 vs 新方案

| 维度 | 旧方案 `main_big_buy.py` | 新方案 `buy_signal.py` |
|------|--------------------------|------------------------|
| 信号源 | 仅 `大笔买入` 一种 | 9种买入 + 9种卖出，全部加权 |
| 买卖对比 | 只看买入，不看卖出 | 净强度 = 买入强度 - 卖出强度 |
| 阈值 | 固定 8000万累计额 | 动态多条件：净强度 + 类型数 + 买卖比 |
| 龙虎榜 | 硬过滤（非游资不买） | 无硬过滤（可选加分） |
| 时间衰减 | 无 | 尾盘信号×1.2，午间×0.8 |
| 信号多样性 | 不检查 | ≥2种不同类型才触发 |
| 价格过滤 | 固定 <100 且 >10 | 无硬限制（由评分自行区分） |
| 创业板/科创板 | 排除 | 包含（同等评分） |
| 输出 | 排序列表 | A/B/C/D 四档分级 |

## 旧方案主要缺陷

### 1. 单一信号易被对倒欺骗
```
旧方案看到：大笔买入(8000万) → 触发买入
实际上：09:34 有大卖盘 + 09:37 大笔买入 → 主力对倒拉高出货
```
新方案会检查**同期卖出强度**，买卖合力为负则过滤。

### 2. 固定金额不适应市值差异
```
8000万对于50亿流通盘是1.6%，对于500亿盘子是0.16%
级别完全不同
```
新方案改用**信号种类数 + 净强度密度**，不依赖绝对金额。

### 3. 龙虎榜硬过滤错过机构行情
```
旧方案：不在龙虎榜 → 不买
但很多强趋势股（如 603738 泰晶科技）的买入信号来自当日主力，
而非前一日游资
```

## 使用方式

```bash
# 默认扫描
python3 pankou_stock_change/scripts/buy_signal.py

# 严格模式：净强度≥50才触发
python3 pankou_stock_change/scripts/buy_signal.py --min-net 50

# 宽松模式：1种信号类型即可
python3 pankou_stock_change/scripts/buy_signal.py --min-types 1

# JSON输出
python3 pankou_stock_change/scripts/buy_signal.py --json
```
